# Latest Updates / News Feed Setup

The website now includes a `Latest Updates` section on the homepage and a dedicated `/updates` page.

The frontend reads posts from:

```txt
public/data/latest-updates.json
```

That file is generated by:

```bash
npm run sync:updates
```

## How it works

1. The sync script reads official social account credentials from `.env` or GitHub repository secrets.
2. It fetches posts from configured official accounts only.
3. It writes the latest 6 normalized posts into `public/data/latest-updates.json`.
4. The React website loads that cached JSON file quickly.
5. If a social API fails, the existing cached feed remains available.

No scraping is used.

## Local setup

Copy the example config:

```bash
cp .env.example .env
```

Fill only the platforms you want to enable:

```env
FACEBOOK_PAGE_ID=
FACEBOOK_ACCESS_TOKEN=

INSTAGRAM_USER_ID=
INSTAGRAM_ACCESS_TOKEN=

LINKEDIN_ORGANIZATION_ID=
LINKEDIN_ACCESS_TOKEN=

X_USERNAME=
X_BEARER_TOKEN=

YOUTUBE_CHANNEL_ID=
```

Run:

```bash
npm run sync:updates
npm run dev
```

## GitHub automatic refresh

The workflow `.github/workflows/sync-social-updates.yml` refreshes the feed every 30 minutes.

Add the same values from `.env.example` as GitHub repository secrets:

Repository → Settings → Secrets and variables → Actions → New repository secret

Required only for the platforms you want:

- `FACEBOOK_PAGE_ID`
- `FACEBOOK_ACCESS_TOKEN`
- `INSTAGRAM_USER_ID`
- `INSTAGRAM_ACCESS_TOKEN`
- `LINKEDIN_ORGANIZATION_ID`
- `LINKEDIN_ACCESS_TOKEN`
- `X_USERNAME`
- `X_BEARER_TOKEN`
- `YOUTUBE_CHANNEL_ID`

## Platform notes

### Facebook

Uses the Facebook Graph API page posts endpoint. The access token must have permission to read the official page posts.

### Instagram

Uses the Instagram Graph API. The Instagram account must be a Business or Creator account connected to a Facebook Page.

### LinkedIn

Uses LinkedIn organization posts. The access token must have organization access for the configured organization ID.

### X

Uses X API v2. By default, retweets/reposts and replies are excluded. Set `SOCIAL_INCLUDE_REPOSTS=true` to include reposts.

### YouTube

Uses the official YouTube channel RSS feed, so no API key is required.

## SEO and performance

This project is currently a Vite React SPA, so true server-side rendering is not available without changing the app architecture.
The implementation still uses semantic HTML, cached JSON, lazy-loaded images, accessible buttons/links, and lightweight browser caching.

For full SSR/Open Graph rendering of each update card, migrate this feature to a server-rendered framework or expose the cached JSON through your hosting platform’s server-rendered layer.
